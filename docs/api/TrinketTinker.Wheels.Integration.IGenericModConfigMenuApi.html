AddComplexOption(IManifest, Func<string>, Action<SpriteBatch, Vector2>, Func<string>?, Action?, Action?, Action?, Action?, Action?, Action?, Func<int>?, string?)

Add an option at the current position in the form using custom rendering logic.

void AddComplexOption(IManifest mod, Func<string> name, Action<SpriteBatch, Vector2> draw, Func<string>? tooltip = null, Action? beforeMenuOpened = null, Action? beforeSave = null, Action? afterSave = null, Action? beforeReset = null, Action? afterReset = null, Action? beforeMenuClosed = null, Func<int>? height = null, string? fieldId = null)

Parameters

mod IManifest

The mod's manifest.

name Func<string>

The label text to show in the form.

draw Action<SpriteBatch, Vector2>

Draw the option in the config UI. This is called with the sprite batch being rendered and the pixel position at which to start drawing.

tooltip Func<string>

The tooltip text shown when the cursor hovers on the field, or null to disable the tooltip.

beforeMenuOpened Action

A callback raised just before the menu containing this option is opened.

beforeSave Action

A callback raised before the form's current values are saved to the config (i.e. before the save callback passed to Register(IManifest, Action, Action, bool)).

afterSave Action

A callback raised after the form's current values are saved to the config (i.e. after the save callback passed to Register(IManifest, Action, Action, bool)).

beforeReset Action

A callback raised before the form is reset to its default values (i.e. before the reset callback passed to Register(IManifest, Action, Action, bool)).

afterReset Action

A callback raised after the form is reset to its default values (i.e. after the reset callback passed to Register(IManifest, Action, Action, bool)).

beforeMenuClosed Action

A callback raised just before the menu containing this option is closed.

height Func<int>

The pixel height to allocate for the option in the form, or null for a standard input-sized option. This is called and cached each time the form is opened.

fieldId string

The unique field ID for use with OnFieldChanged(IManifest, Action<string, object>), or null to auto-generate a randomized ID.

Remarks

The custom logic represented by the callback parameters is responsible for managing its own state if needed. For example, you can store state in a static field or use closures to use a state variable.

AddPage(IManifest, string, Func<string>?)

Start a new page in the mod's config UI, or switch to that page if it already exists. All options registered after this will be part of that page.

void AddPage(IManifest mod, string pageId, Func<string>? pageTitle = null)

Parameters

mod IManifest

The mod's manifest.

pageId string

The unique page ID.

pageTitle Func<string>

The page title shown in its UI, or null to show the pageId value.

Remarks

You must also call AddPageLink(IManifest, string, Func<string>, Func<string>?) to make the page accessible. This is only needed to set up a multi-page config UI. If you don't call this method, all options will be part of the mod's main config UI instead.
